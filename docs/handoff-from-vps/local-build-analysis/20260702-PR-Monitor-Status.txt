    5. Quality concerns

    Primary quality posture:
    The project has correctly shifted from “make crawler work” to “no silent failures.”

    Quality target:
    - 95%+ actionable classification
    - 90%+ usable content initially, path to 95%
    - 0 silent failures

    Known crawl baseline:
    - 25 URLs attempted
    - 18 PASS
    - 6 BLOCKED
    - 1 PARTIAL
    - 100% actionable classification
    - 72% usable content
    - 0 silent failures

    Interpretation:
    Classification quality is strong. Usable crawl coverage is not yet strong enough to safely push full runs downstream without fallback/recovery.

    Main quality risks:
    - Bot blocking / Cloudflare / 403s
    - JS/lazy-loaded pages giving partial content
    - Dates and CFP deadlines being wrong or stale
    - CFP info hidden on subpages
    - Platform false positives like Sessionize
    - Past-event rollover to next year       2h 4m │ ⏲ 1m 35s │ ✓ 2h 1m
    - Human-verified corrections being overwritten by lower-confidence recrawls
    - Customer trust risk if freshness/provenance/status are unclear

    Extraction/model benchmark status:
    - Top free-model benchmark exists.
    - openrouter/owl-alpha ranked highest structurally in one multi-page evidence benchmark.
    - But docs explicitly say not to choose final default until gold-set accuracy review is complete.

    6. Key product risk

    The risk is not “we don’t have a product.”

    The risk is:
    We have a useful product spine, but if automation and quality gates are not tightened, Nicolia may still experience it as another tool that requires human babysitting.

    That would keep Matt/the team as the constraint.

    7. What this means strategically

    PR Monitor is still a GO project.

    Why:
    - Paying customer.
    - Clear pain.
    - Strong domain fit for Matt.
    - Working artifact already exists.
    - Expansion path is obvious: alerts, weekly reports, client-fit scoring, awards/media opportunities, PR intelligence.

    But the next phase should not chase broad features first.

    Recommended next focus:
    1. Finish quality gate loop for Crawl → Extract → Enrich → Review.
    2. Prove local/browser fallback improves BLOCKED/PARTIAL URL coverage.
    3. Complete gold-set scoring before trusting extraction model defaults.
    4. Wire M6/M7: scheduled runs + alerts + executive report.
    5. Only then expand into client-fit scoring, awards/media, SQL/Sheets integration.
